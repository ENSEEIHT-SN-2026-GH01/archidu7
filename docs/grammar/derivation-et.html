Lecture rapide

• Profondeur 4→8 : décomposition de la ParamList a, b : c. Le séparateur : n'est pas distingué de , au niveau grammaire (sémantique attachée plus tard).

• Profondeur 6→13 : chemin d'expression SOTC → SOT → Term → Factor * Factor. Le produit a * b apparaît via TR → '*' Factor TR, puis TR → ε.

• Feuilles (de gauche à droite) restituent exactement la séquence de tokens : module ET ( a , b : c ) c = a * b end module.

• Nœuds ε : productions vides. Chaque non-terminal optionnel (queue de liste, tail de signal, tri-state tail) se ferme par un ε.
