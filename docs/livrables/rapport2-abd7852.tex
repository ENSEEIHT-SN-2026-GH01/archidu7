\section*{Contexte et objectif de l'it\'eration}

Le projet long SHDL vise un simulateur de circuits logiques avec interface JavaFX. L'\'equipe compte neuf personnes r\'eparties sur six axes (\'editeur, parser, automates, simulateur, IHM, int\'egration). Ma responsabilit\'e demeure la \textbf{structure g\'en\'erale} et l'\textbf{interpr\'etation du SHDL} (ticket \textsc{Scrum-23}). L'it\'eration~1 avait pos\'e un parser \`a descente r\'ecursive vers un AST. L'objectif de l'it\'eration~2 \'etait double~: (i)~remplacer ce parser par une version \emph{table-driven} stricte, conforme \`a l'enseignement de compilation, et (ii)~brancher concr\`etement le parser au simulateur amont via une couche de conversion. Ces deux objectifs ont \'et\'e atteints et fond\'es sur 82~commits.

\section*{Activit\'es r\'ealis\'ees}

\begin{itemize}
  \item \textbf{Refonte compl\`ete du parser en LL(1) table-driven} (\texttt{parser/ll1/tabledriven/})~: construction de la table d'analyse $M[NT, T]$ \`a partir des ensembles \textsc{First} et \textsc{Follow} (\texttt{TableBuilder}, lookup en $O(1)$), driver \`a pile g\'en\'erique sans \emph{switch} par non-terminal (\texttt{CstParser}). L'ancien parseur r\'ecursif est conserv\'e en branche \texttt{feature/parser-ll1-old} mais marqu\'e obsol\`ete.
  \item \textbf{Concrete Syntax Tree} (CST) en remplacement de l'AST~: types \texttt{sealed} \texttt{CstNode = CstLeaf | CstInternal} (records immuables), m\'ethodes de navigation \texttt{first(Symbol)} / \texttt{allOf(Symbol)} / \texttt{has(Symbol)} / \texttt{startOffset()} / \texttt{endOffset()}. Trivia (whitespace, commentaires) filtr\'es avant parsing, offsets pr\'eserv\'es. Outil de d\'ebogage \texttt{CstDumper} avec sortie indent\'ee.
  \item \textbf{Conversion CST $\to$ \texttt{simulateur.Module}} (\texttt{parser/conversion/}, sp\'ecification \texttt{docs/specs/2026-05-06-cst-conversion-design.md})~: couche \emph{int\'egration} qui produit l'IR du simulateur de Mati \`a partir du CST. P\'erim\`etre S1~: \textbf{combinatoire scalaire stricte}. Les hors-scope (vecteurs, concat\'enation, m\'emoire, appel de module, litt\'eraux en RHS, doublons) sont rejet\'es par une \texttt{ConversionException} typ\'ee (sept raisons \'enum\'er\'ees, offset et type de n\oe ud port\'es).
  \item \textbf{Bug fix amont}~: \texttt{simulateur.Erwan.Erwan.OR()} retournait \texttt{Op.AND}. Corrig\'e dans le commit \texttt{37cc8d8} avec test sentinelle anti-r\'egression.
  \item \textbf{Documentation pipeline}~: \texttt{README.md} de la branche r\'e\'ecrit pour exposer en trois lignes le pipeline destin\'e au bundle JavaFX~:
\begin{verbatim}
  CstNode cst   = CstParser.parse(shdlSource);
  Module module = Conversion.convert(cst);
  FileSimulateur sim = new FileSimulateur(module.Plan);
\end{verbatim}
  \item \textbf{Sp\'ecifications et plans} versionn\'es~: \texttt{docs/specs/2026-04-30-ll1-table-driven-cst-design.md}, \texttt{docs/specs/2026-05-06-cst-conversion-design.md}, \texttt{docs/plans/} associ\'es.
\end{itemize}

\section*{R\'esultats}

\begin{itemize}
  \item \textbf{193 tests JUnit verts} (107 au sprint~1, soit \textbf{+86 tests}), r\'epartis sur 39~classes. Couverture~: parser CST (offsets, $\varepsilon$-productions, erreurs avec position v\'erifi\'ee, multi-instance), conversion (cas nominaux scalaires \texttt{ET}/\texttt{OU}/\texttt{NOT}, rejets exhaustifs hors S1 avec offsets), bout-en-bout (\texttt{xor} table de v\'erit\'e 4~lignes via \texttt{FileSimulateur}, fixtures \texttt{.shdl} sur disque conformes \`a la sp\'ec.).
  \item \textbf{14~fichiers Java} ajout\'es sous \texttt{parser/conversion/} et \texttt{parser/ll1/tabledriven/} (hors tests).
  \item Une \textbf{int\'egration r\'eussie} avec le code amont d'Erwan (\texttt{parser/lexer/} via \texttt{CstParser.parse}) et de Mati (\texttt{simulateur.Module}, \texttt{FileSimulateur}). Le pi\`ege \texttt{TERM\_REST} relev\'e au sprint~1 dispara\^it~: la grammaire LL(1) est gel\'ee par \texttt{GrammarFreezeTest} et v\'erifi\'ee sans conflit \textsc{First}/\textsc{First} ou \textsc{First}/\textsc{Follow}.
  \item \textbf{\'Ecarts assum\'es vis-\`a-vis de la grammaire amont} (\texttt{shdl\_grammar\_LL1\_v2.txt}) document\'es dans \texttt{parser/ll1/tabledriven/README.md}~: typo \texttt{Factor} parens, NTs morts retir\'es, casse uniformis\'ee, op\'erateur m\'emoire \texttt{::=} corrig\'e en \texttt{:=}.
\end{itemize}

\section*{Difficult\'es et r\'eussites}

La principale difficult\'e a \'et\'e d'absorber les \emph{instabilit\'es du code amont}~: la branche \texttt{interpretation} d'Erwan a livr\'e des stubs successifs, et le bug \texttt{Erwan.OR()} cit\'e ci-dessus n'a \'et\'e d\'etect\'e que par un test e2e sur une table de v\'erit\'e \texttt{XOR}. La discipline TDD --- \'ecriture du test n\'egatif avant le passage --- a \'et\'e pay\'ee plusieurs fois ce sprint.

La r\'eussite principale est la \textbf{s\'eparation stricte des r\^oles} entre parser, CST, conversion et IR simulateur. Le CST sert d'interface contractuelle~: la grammaire peut \'evoluer sans casser la conversion tant que les m\'ethodes de navigation tiennent. La conversion peut grandir vers le s\'equentiel ou les vecteurs sans toucher au parser. Cette d\'ecoupe a permis de para\-ll\'eliser quatre vagues d'agents de v\'erification automatique (revue de spec, revue d'impl\'ementation, smoke tests, tests d'int\'egration) sans pollution du contexte.

Limitation honn\^ete~: le scope S1 est \emph{volontairement} pauvre. Le s\'equentiel (bascules, \texttt{:=}), les vecteurs, les appels de module sont accept\'es par le parser mais refus\'es par la conversion. L'extension est m\'ecanique mais n\'ecessitera un IR enrichi c\^ot\'e simulateur (pas \`a ma main).

\section*{Manuel utilisateur}

\textbf{Pr\'e-requis}~: JDK~25, JUnit 4.13.2 et Hamcrest 1.3 (fournis dans \texttt{lib/}).

\textbf{Compiler et lancer la batterie de tests}~:
\begin{verbatim}
  ./build.sh test     # compile parser + conversion + simulateur + tests
  ./run.sh            # 193 tests JUnit
\end{verbatim}

\textbf{Convertir un programme SHDL en circuit pr\^et \`a simuler}~:
\begin{verbatim}
  import parser.ll1.tabledriven.CstParser;
  import parser.conversion.Conversion;
  import simulateur.Module;
  import simulateur.FileSimulateur;

  CstNode cst        = CstParser.parse(shdlSource);     // ParsingException possible
  Module  module     = Conversion.convert(cst);         // ConversionException possible
  FileSimulateur sim = new FileSimulateur(module.Plan); // implements Simulateur
\end{verbatim}

Les \texttt{Exception} expos\'ees portent l'\texttt{offset()} dans la source, le contexte syntaxique (\texttt{contextPath()} c\^ot\'e parser, \texttt{nodeKind()} c\^ot\'e conversion) et un code d'erreur structur\'e, exploitables directement par l'IHM pour le surlignage.

Le code source est publi\'e sur la branche \texttt{feature/parser-ll1-table-driven}. La sp\'ecification d\'etaill\'ee de la conversion se trouve dans \texttt{docs/specs/2026-05-06-cst-conversion-design.md}.
